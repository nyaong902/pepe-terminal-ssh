## 1. 시작하기

### 설치

[GitHub Releases](https://github.com/HyungdukSeo/pepe-terminal-ssh/releases) 에서 최신 Windows 설치 파일을 내려받습니다:

- **`PePe Terminal(SSH) Setup X.Y.Z.exe`** — NSIS 설치 파일 (일반적 설치)

> ⚠️ 코드 서명이 없어 Windows SmartScreen 경고가 뜰 수 있습니다. "추가 정보 → 실행"으로 진행하세요.

## 2. 세션 관리

### 세션 리스트

![세션 리스트](screenshots/02-session-list.png)

- 세션은 폴더별로 계층 구조로 정리 가능
- 드래그앤드롭으로 폴더 이동
- 우클릭으로 이름 변경/삭제

### 선택 방식

- **단일 클릭**: 세션 선택 (인코딩/설정 바 표시)
- **더블 클릭**: 현재 선택된 패널에 연결
- **Ctrl + 클릭**: 다중 선택 토글
- **Shift + 클릭**: 범위 선택 (anchor부터 클릭 위치까지)

### 다중 선택 + 일괄 연결

여러 세션을 선택한 뒤 우클릭하면 다음 옵션이 나옵니다:

![다중 선택 컨텍스트 메뉴](screenshots/03-multi-select-ctx.png)

- **📑 미니탭으로 연결** — 한 패널 안에 탭 형태로 모두 추가
- **▐ 세로 분할로 연결** — 좌우 2분할 (여러 개면 N-열)
- **▄ 가로 분할로 연결** — 위아래 2분할 (여러 개면 N-행)
- **⊞ 타일 분할로 연결** — 자동 그리드 배치
  - N=2 → 1×2, N=3 → 2행(위2/아래1), N=4 → 2×2, N=6 → 2×3, N=9 → 3×3

## 3. 세션 편집기

![세션 편집기 — 기본](screenshots/04-session-editor-basic.png)

### 기본 필드

| 필드 | 설명 |
|---|---|
| **이름** | 세션 표시 이름 |
| **호스트** | IP 또는 도메인 |
| **포트** | 기본 22 |
| **사용자** | SSH 로그인 계정 |
| **인증** | 비밀번호 / 키 파일 |
| **인코딩** | utf-8 / cp949 / euc-kr 등 |
| **폴더** | 세션 리스트에서의 분류 |
| **파일 트리 초기 경로** | `Ctrl+Shift+E` 로 열었을 때 기본 표시 경로 (비우면 홈) |

### 점프 호스트 (ProxyJump) — v2.0.2+

![세션 편집기 — 점프 호스트](screenshots/05-session-editor-jump.png)

bastion 서버 뒤에 있는 내부 서버에 접근하려면 점프 타겟 설정:

| 필드 | 설명 |
|---|---|
| **점프 타겟 호스트** | 최종 연결할 내부 서버 (예: `I-MPM01`) |
| **점프 타겟 사용자** | 예: `root` |
| **점프 타겟 포트** | 기본 22 |
| **점프 타겟 비밀번호** | (선택) 비우면 primary 의 `~/.ssh/id_rsa` 자동 사용 |

**동작 방식**: 앱 → primary 호스트(bastion) → TCP 터널 → 점프 타겟에 직접 SSH + SFTP. 터미널·파일트리 모두 점프 타겟 기준으로 동작.

**Solaris 같은 구형 sshd** 도 레거시 알고리즘(`diffie-hellman-group1-sha1`, `ssh-dss` 등) 자동 협상.

### 로그인 스크립트 (Expect/Send)

![세션 편집기 — 로그인 스크립트](screenshots/06-session-editor-login-script.png)

접속 직후 자동 입력할 스크립트 (예: 2단계 인증 통과, 자동 alias 실행):
- **Expect**: 화면에서 기다릴 문자열 (정규식 지원)
- **Send**: 매칭 시 전송할 문자열

## 5. 분할 화면

### 분할 실행

- **패널 헤더의 ▐ / ▭ 버튼**
- **단축키**: `Ctrl+Shift+→` (세로), `Ctrl+Shift+↓` (가로) 등 (단축키 섹션 참고)
- **창 메뉴**: 메뉴바의 "창 → 세로 분할 / 가로 분할"

### 세션 선택 팝업

분할 버튼/단축키 누르면 **현재 활성 세션의 폴더 내 모든 세션** 리스트가 뜹니다:

![분할 세션 선택](screenshots/09-split-session-picker.png)

선택 시 새 터미널이 그 세션으로 연결되며 분할 패널로 배치됩니다. 취소 시 빈 분할.

### 분할 리사이즈

분할 경계선을 마우스로 드래그하여 크기 조절.

## 8. 파일 트리 (Ctrl+Shift+E)

### 열기 / 토글

![파일 트리](screenshots/12-file-tree.png)

단축키 **`Ctrl+Shift+E`** 로 현재 활성 터미널의 원격 파일 트리를 패널 내부 좌측에 표시/숨김.

OSC 7 기반으로 **현재 디렉토리 자동 추적** — 터미널에서 `cd`하면 파일트리 기준 경로도 따라갑니다.

### 경로바

![경로바](screenshots/13-file-tree-pathbar.png)

| 버튼 | 기능 |
|---|---|
| 경로 입력 + Enter | 지정 경로로 이동 |
| **↵** | 입력창 내용으로 이동 |
| **⬆** | **현재 경로에 파일 업로드** (다중선택 가능) — v2.0.3 |
| **⟳** | 현재 경로 새로고침 (stale 방지) |

### 파일/폴더 클릭 동작 — v2.0.3+

- **파일 단일 클릭**: 선택(파란 강조). 에디터 오픈 안 됨.
- **파일 더블 클릭**: 에디터에서 파일 열기
- **폴더 단일 클릭**: 확장/축소 토글
- **Ctrl + 클릭**: 다중 선택 토글
- **Shift + 클릭**: 범위 선택

## 10. 파일 전송 탭

### 탭 생성

![파일 전송 탭](screenshots/18-file-transfer-tab.png)

두 가지 방법:

1. **세션 리스트에서 우클릭 → 📁 파일 전송**
2. **탭 바의 + → 📁 파일 전송 워크스페이스**

### 2패널 구조

![파일 전송 2패널](screenshots/19-file-transfer-dual.png)

좌/우 각각 독립적인 파일 소스 (로컬 / 원격 SFTP).

**소스 드롭다운**에서 선택 (v2.0.3+ 모든 세션 표시):
- `🖥️ 로컬` — PC의 파일시스템
- `🟢 <세션명> [폴더]` — 터미널로 연결된 세션 (기존 SSH 재사용)
- `⚪ <세션명> [폴더] (호스트)` — 세션리스트에만 있고 미연결 — **선택 시 백그라운드 SFTP 자동 연결** (점프 호스트 설정도 반영)
- `🔌 <사용자>@<호스트>` — 수동 SFTP 직접 연결 결과
- `🔌 SFTP 직접 연결...` — 임시 연결 추가 다이얼로그

### 점프 호스트 자동 지원 — v2.0.2+

세션에 점프 타겟이 설정돼 있으면, 파일 전송에서도 **자동으로 ProxyJump 로 내부 서버에 연결**. 미연결 세션을 드롭다운에서 고르기만 해도 점프 포함 연결이 자동 수행됩니다.

### 전송

- 좌→우 또는 우→좌 방향 전송 버튼
- 드래그앤드롭 지원
- 다중 파일 선택 후 일괄 전송
- 진행률 표시 + 완료 후 자동 새로고침

## 11. 일괄 텍스트 전송

![일괄 전송바](screenshots/20-broadcast-bar.png)

단축키 (설정 가능, 기본: `Ctrl+Shift+B`) 로 토글.

### 대상 선택 드롭다운

| 범위 | 의미 |
|---|---|
| **보이는 탭** (기본) | 현재 활성 워크스페이스의 모든 세션 |
| **현재 세션** | 현재 포커스된 세션 하나 |
| **연결된 세션** | 모든 워크스페이스의 연결된 세션 전체 |

### 전송 버튼

- **전송** — Enter 키와 동일 (입력한 텍스트 전송)
- **📤 파일전송** — 일괄 파일 전송 모달 오픈 (아래 섹션)
- **↑** — ↑ 방향키 (쉘 히스토리 이전 명령)
- **↓** — ↓ 방향키 (히스토리 다음 명령) — v2.0.3+
- **^C** — Ctrl+C (SIGINT)
- **^D** — Ctrl+D (EOF)

### 히스토리

입력창 우측 **▾** 로 전송 이력 드롭다운. 위 방향키로도 불러오기.

### 사용 흐름

1. **대상 세션** 선택 (보이는 탭 / 연결된 세션)

5. 실시간 로그로 세션별 성공/실패 확인

### 원격 파일 소스 — v2.0.3+

![원격 파일 picker](screenshots/22-remote-file-picker.png)

"+ 원격 파일 (다른 서버)" 클릭 시 picker 모달:

- **소스 세션 드롭다운** — **전체 세션 리스트** 표시 (sessions.json 기준)
  - 🟢 현재 터미널로 연결된 세션 (기존 SSH 재사용)
  - ⚪ 미연결 세션 (선택 시 **백그라운드 SFTP 자동 연결**. 점프 호스트도 ProxyJump)
  - 연결 중엔 "연결 중..." 노란색 표시
- **경로바** — 경로 입력 + Enter / ▲ 상위 / ⟳ 새로고침
- **파일 리스트** — 클릭으로 다중 선택, 폴더 더블클릭 진입
- **선택 항목 추가** — 일괄 전송 목록에 🌐 아이콘으로 표시. 모달은 닫히지 않고 다른 세션에서 연속 추가 가능
- 모달 닫을 때 임시 SFTP 연결 자동 해제 (`feSftpDisconnect`)

## 13. ProxyJump (점프 호스트)

![점프 호스트 구조](screenshots/24-proxyjump-diagram.png)

### 언제 필요한가

bastion / jump host 뒤에 있는 내부 서버에 접근할 때. 예:
- 사용자 PC → **EMS (bastion)** → **MPM01 (내부)**
- 사용자 PC → **공인 서버** → **사설망 내부 서버**

### 설정 방법

세션 편집기의 **점프 타겟** 필드에 내부 서버 정보 입력 (섹션 3 참고).

### 동작 세부

1. primary (EMS) 로 일반 SSH 연결
2. primary 위에 `forwardOut` TCP 터널 생성 → 점프 타겟 22번
3. 터널 위에서 **두 번째 SSH 핸드셰이크** (점프 타겟에 직접)
4. 이후 **터미널·SFTP·파일전송·파일트리 모두 점프 타겟 기준**
5. 인증은 (1) 비밀번호 입력 시 사용, (2) 비었으면 primary 의 `~/.ssh/id_rsa` 를 자동 SFTP 로 읽어 재사용 → 기존 passwordless 설정 그대로 활용

### Solaris 등 레거시 서버

`diffie-hellman-group1-sha1`, `ssh-dss`, `3des-cbc` 같은 구형 알고리즘까지 자동 협상.

### 동시 다수 접속

- **500ms stagger** 로 bastion 의 `MaxStartups` drop 방지
- **20s watchdog + 3회 자동 재시도** — 일시적 실패 자동 복구
- Solaris 는 KEX 연산이 느려 7~8초 소요될 수 있음 (정상)

## 16. 트러블슈팅

### "Connection lost before handshake" 또는 JavaScript error dialog

v2.0.3 부터 수정됨. 구버전이면 업데이트 권장. 증상 지속 시 DevTools Console(개발 모드) 에 `[ssh-error]` 또는 `[jump-xxxxx]` 로그 확인.

### 동시 다수(13+) Solaris 점프 접속 시 일부 실패

Solaris sshd 의 MaxStartups 또는 KEX CPU 경합. 현재 대응:
- 500ms stagger
- 20초 watchdog + 5초 간격 3회 자동 재시도
- 그래도 실패하면 한 번에 띄우는 세션 수를 줄이거나 간격을 둬서 접속

### "Unsupported algorithm" 에러

Node/OpenSSL 빌드에 해당 암호 알고리즘 바인딩 없음. v2.0.2+ 에서 시스템 가용 알고리즘만 자동 필터하도록 수정.

### 점프 연결 되는데 파일 전송 메뉴에서 다른 서버 파일이 보임

세션 편집 창에서 **점프 타겟 필드가 실제로 저장됐는지** 확인. 세션 저장 후 다시 파일 전송 시도.

### DevTools 열기

개발 모드(`npm run dev`) 에선 자동 오픈. 설치 빌드에선 기본 미노출 (개발자 전용 기능).
